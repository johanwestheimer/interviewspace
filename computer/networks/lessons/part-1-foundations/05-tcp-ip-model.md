## 0. Откуда взялась TCP/IP

Небольшой исторический контекст, потому что он объясняет, почему модель именно такая.

- OSI (Урок 4) разрабатывалась комитетами ISO «сверху» как красивый теоретический стандарт.
- TCP/IP родилась «снизу», из практики — из проекта **ARPANET** (предшественник интернета, конец 1970-х, работы Винта Серфа и Боба Кана). Сначала был работающий код и реальные сети, а уже потом — описание.

🔑 И в этом вся суть: **TCP/IP победила, потому что она работала и была проще.** Пока комитеты OSI годами согласовывали идеальную 7-уровневую модель, инженеры уже гоняли реальный трафик по TCP/IP. Это хороший урок и для системного дизайна: *работающая простая вещь почти всегда побеждает идеальную, но сложную*.

> Из-за этой истории модель называют по-разному: «модель TCP/IP», «модель DoD» (Department of Defense), «интернет-модель». Это всё одно и то же.

## 1. Четыре уровня TCP/IP

Классическая модель TCP/IP имеет **4 уровня** (снизу вверх):

| # | Уровень TCP/IP | За что отвечает | Ключевые протоколы |
|---|----------------|-----------------|--------------------|
| 4 | **Application** (прикладной) | Всё, что нужно приложениям: формат, сессия, данные | HTTP, DNS, TLS, SSH, SMTP, FTP, DHCP |
| 3 | **Transport** (транспортный) | Доставка нужному приложению (порты), надёжность | TCP, UDP, QUIC |
| 2 | **Internet** (межсетевой) | Логическая адресация и маршрутизация через сети | IP (IPv4/IPv6), ICMP |
| 1 | **Link** (канальный/сетевого доступа) | Передача в пределах одной физической сети | Ethernet, Wi-Fi, ARP |

> 💡 **Названия «плавают».** Нижний уровень называют по-разному: *Link*, *Network Access*, *Network Interface*. Верхний — просто *Application*. Не пугайся разночтений в учебниках — суть одна.

### Краткий разбор каждого

- **Link (1).** «Как передать данные железному соседу по конкретной физической сети». Сюда свалено всё про кадры, MAC-адреса, физическую передачу. Заметь: TCP/IP **не детализирует** этот уровень — ей всё равно, Ethernet там, Wi-Fi или что-то ещё. Это и есть модульность из Урока 3 в действии.
- **Internet (2).** Сердце модели. Здесь живёт **IP** — протокол, который даёт глобальные адреса и доставляет пакеты через множество сетей (best-effort, без гарантий — Урок 2). Слово «Internet» здесь = «между сетями».
- **Transport (3).** Здесь возвращаются гарантии (сквозная нить курса!). **TCP** — надёжно и по порядку; **UDP** — быстро и без обязательств. Адресация приложений через порты.
- **Application (4).** Всё, что выше транспорта. Заметь: TCP/IP **схлопнула** три верхних уровня OSI (Application + Presentation + Session) в один. На практике приложение само разбирается с форматом, шифрованием и сессиями.

## 2. Сопоставление OSI ↔ TCP/IP (это надо знать наизусть)

Главная картинка урока. Запомни её — это прямой вопрос на собесе.

```
        OSI (7 уровней)                 TCP/IP (4 уровня)
   ┌─────────────────────┐         ┌──────────────────────┐
 7 │   Application        │ ┐       │                      │
 6 │   Presentation       │ ├──────►│   Application        │  (HTTP, DNS, TLS, SSH)
 5 │   Session            │ ┘       │                      │
   ├─────────────────────┤         ├──────────────────────┤
 4 │   Transport          │ ───────►│   Transport          │  (TCP, UDP)
   ├─────────────────────┤         ├──────────────────────┤
 3 │   Network            │ ───────►│   Internet           │  (IP, ICMP)
   ├─────────────────────┤         ├──────────────────────┤
 2 │   Data Link          │ ┐       │                      │
 1 │   Physical           │ ┴──────►│   Link               │  (Ethernet, Wi-Fi, ARP)
   └─────────────────────┘         └──────────────────────┘
```

Запомни три «склейки»:

1. **OSI 7+6+5 (Application+Presentation+Session) → один Application** в TCP/IP.
2. **OSI 4 (Transport) ↔ Transport** — один в один.
3. **OSI 3 (Network) ↔ Internet** — один в один (только название другое).
4. **OSI 2+1 (Data Link+Physical) → один Link** в TCP/IP.

> 💡 Середина (Transport и Network/Internet) совпадает идеально — это «талия» обеих моделей. Различия только по краям: сверху TCP/IP проще (один уровень вместо трёх), снизу — объединяет два в один.

### А что за «5-уровневая модель»?

Часто в учебниках (и на собесах) встречается **гибридная 5-уровневая модель**: она берёт 4 уровня TCP/IP, но **разделяет** нижний Link обратно на **Physical** и **Data Link** (как в OSI). Получается: Physical → Data Link → Network → Transport → Application. Это удобный компромисс для обучения. Если на собесе рисуешь 5 уровней — это нормально, просто оговорись, что это гибрид.

## 3. Почему TCP/IP, а не OSI: суть для собеседования

| Критерий | OSI | TCP/IP |
|----------|-----|--------|
| Происхождение | теория, комитеты ISO | практика, ARPANET, рабочий код |
| Когда появилось | модель → потом протоколы (которые не взлетели) | протоколы → потом описание |
| Уровней | 7 | 4 (или 5 в гибриде) |
| Где используется | как **словарь/эталон** для обучения и обсуждения | как **реальный стек** интернета |
| Гибкость нижних уровней | строго Physical + Data Link | один Link, не диктует технологию |

🔑 **Вывод:** OSI — это **линейка и словарь** (удобно говорить «проблема на L2»), TCP/IP — **реальная конструкция**. На практике инженеры мыслят в TCP/IP, но *описывают* проблемы в терминах номеров уровней OSI. Поэтому обе модели нужны вместе.

## 4. Как стек работает на реальном примере (предпросмотр)

Когда ты открываешь сайт, данные проходят сверху вниз через стек TCP/IP на твоём компьютере:

```
[Application]  Браузер формирует HTTP-запрос "GET /"
       │ (добавит TLS-шифрование)
[Transport]   TCP оборачивает в сегмент: + порт назначения 443, seq-номера
       │
[Internet]    IP оборачивает в пакет: + IP-адрес google.com и твой IP
       │
[Link]        Ethernet/Wi-Fi оборачивает в кадр: + MAC-адрес роутера
       │
       ▼  биты в провод/радио → ... сеть ... → у сервера всё разворачивается снизу вверх
```

Каждый уровень добавил свой «конверт» (заголовок). Это **инкапсуляция** — и ей целиком посвящён следующий, финальный урок Части I. Здесь же запомни главное: реальные данные путешествуют именно по этим четырём уровням TCP/IP.

## 5. Соберём картину воедино

- **TCP/IP** — реальная модель интернета, родилась из практики (ARPANET) и победила OSI простотой и работоспособностью.
- **4 уровня:** Link (Ethernet/Wi-Fi/ARP) → Internet (IP, ICMP) → Transport (TCP, UDP) → Application (HTTP, DNS, TLS, SSH).
- **Сопоставление с OSI:** Application TCP/IP = L7+L6+L5 OSI; Transport = L4; Internet = L3; Link = L2+L1.
- Середина (Transport + Network/Internet) у обеих моделей совпадает; различия по краям.
- Существует гибридная **5-уровневая** модель (Link разбит на Physical + Data Link) — для удобства обучения.
- На практике: думаем в TCP/IP, *обсуждаем* в номерах уровней OSI.

## 6. Как это спрашивают на собеседовании

- **«Сравните OSI и TCP/IP».** → Назвать 4 уровня TCP/IP, дать сопоставление (склейки 7/6/5→App и 2/1→Link), сказать про происхождение и почему TCP/IP победила.
- **«Сколько уровней в TCP/IP?»** → 4 (классика) или 5 (гибрид с разделённым Link). Оговорить разницу — плюс.
- **«На каком уровне TCP/IP работает IP / TCP / HTTP / Ethernet?»** → Internet / Transport / Application / Link.
- **«Почему интернет использует TCP/IP, а не OSI?»** → Простота, работающий код раньше стандарта, гибкость нижнего уровня. Хороший момент про «работающее простое побеждает идеальное сложное».
- **«Куда в TCP/IP делись Session и Presentation?»** → Схлопнулись в Application; их функции (формат, шифрование, сессии) выполняет приложение/TLS.

> 🎯 **Уровень топ-компаний:** показать, что ты держишь в голове **обе** модели и понимаешь их роли — OSI как словарь, TCP/IP как реальность — и свободно переводишь между ними. Это маркер зрелого инженера.
